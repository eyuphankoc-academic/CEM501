### Scenario

You are the assistant PM on the **Kadikoy Bridge Rehabilitation Project** (Contract #BR-2024-071). At the end of the day on March 14, 2026, you receive field notes from multiple people on site — in different formats, with overlapping and sometimes conflicting details:

| File | Source | Format |
|------|--------|--------|

| `02_hasan_beton_plus.txt` | Hasan Eren (Foreman, Beton Plus) | WhatsApp message |
| `03_mehmet_kaya_steel.txt` | Mehmet Yildiz (Foreman, Kaya Steel) | Handwritten note |
| `04_elif_project_engineer.txt` | Elif Sahin (Project Engineer) | Email |
| `05_safety_log_entry.txt` | T. Aksoy (acting Safety Officer) | Safety log page |

| `10_municipality_voicemail.txt` | Canan (IBB Infrastructure) | Voicemail transcription |
| `11_night_watchman.txt` | Yusuf Karaca (Night Security) | Night security report |

These files are in the course repository at `Week06_Reports_Summarization/field_notes/`.

**Note:** Some sources report the same events with slightly different details (times, quantities). Part of the exercise is deciding which source is most authoritative and reconciling discrepancies.

---

### Your Task

You will build this in two steps:

#### Step 1: Write a script that generates an LLM draft report

Write a Python script (`generate_report.py`) that:

1. Reads all 11 field note files from the `field_notes/` folder
2. Sends them to an LLM via API (Anthropic, OpenAI, or Google — your choice) with a prompt that instructs the model to produce a daily construction report
3. Your prompt should specify the report format from Part II of the lecture (the seven required elements) and instruct the model to reconcile conflicting details by preferring official documents (delivery tickets, safety logs) over informal notes
4. Saves the LLM-generated draft to a file (e.g., `draft_report.txt`)

You may use Claude Code, Codex CLI, or Gemini CLI to help you write this script.

#### Step 2: Review, correct, and finalize

Read the LLM-generated draft carefully. Then produce your **final report** by:

- Fixing any hallucinated details (information the LLM added that is not in any source file)
- Fixing any omissions (information from the source files that the LLM left out)
- Correcting any discrepancies where the LLM picked the wrong source
- Ensuring the tone is objective and factual — suitable for a business record

This is the core principle from the lecture: **LLM summaries are drafts, not records.**

Submit the following to the course repository under `Week06_Reports_Summarization/`:

### Report Requirements

Your final report must follow the template from Section 2.2 of the lecture and include all seven required elements:

1. **Date, project name, report number**
2. **Weather** (morning + afternoon)
3. **Manpower** (by trade, subcontractor, count)
4. **Equipment** (status: active, idle, standby)
5. **Work performed** (locations, quantities, %)
6. **Delays, issues, RFI status**
7. **Safety and visitors**
